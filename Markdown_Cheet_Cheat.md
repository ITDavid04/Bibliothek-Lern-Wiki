

# ✍️ Schreib-Guide & Tools

Damit unser Wiki professionell aussieht und leicht zu pflegen ist, nutzen wir **Markdown**. Hier findest du eine kurze Übersicht der wichtigsten Befehle und nützliche Programme für Linux.

## 🛠 Empfohlene Programme (Linux)

Da wir im Linux-Umfeld arbeiten, gibt es hervorragende Editoren, die uns das Schreiben erleichtern:

*   **ReText:** Ein sehr schlanker, Open-Source-Editor für Linux. Er bietet eine Live-Vorschau (Split-Screen) und ist perfekt für schnelles Arbeiten ohne viel Schnickschnack.
*   **Obsidian:** Ideal, wenn du deine Notizen vernetzen willst. Es nutzt lokal gespeicherte Markdown-Dateien und ist extrem mächtig für die Wissensorganisation.
*   **VS Code (Visual Studio Code):** Der Standard für viele Entwickler. Mit der Tastenkombination `Strg` + `Shift` + `V` öffnest du direkt eine Live-Vorschau deiner Markdown-Datei.
*   **GitHub Web-Editor:** Für kleine Änderungen kannst du einfach die Taste `.` (Punkt) drücken, während du im Repository bist – GitHub öffnet dann einen webbasierten Editor (VS Code light).

## 📝 Markdown Kurzanleitung

### Überschriften
```markdown
# Hauptüberschrift (nur 1x pro Datei)
## Sektion
### Untersektion
```

### Textformatierung
*   **Fett:** `**Dein Text**`
*   *Kursiv:* `*Dein Text*`
*   ~~Durchgestrichen:~~ `~~Text~~`
*   `Code-Zeile`: `` `Code hier` ``

### Listen
```markdown
*   Punkt A
*   Punkt B
    *   Unterpunkt

1. Erster Schritt
2. Zweiter Schritt
```

### Links & Bilder
*   **Link:** `[Titel des Links](https://beispiel.de)`
*   **Bild:** `![Beschreibung](bild_url_oder_pfad)`

### Infoboxen (GitHub Spezial)
```markdown
> [!NOTE]
> Hilfreiche Info für das Team.

> [!TIP]
> Ein praktischer Trick für das Lernfeld.

> [!CAUTION]
> Achtung: Hier bitte nichts ohne Rücksprache löschen!
```

## 💡 Profi-Tipps für unser Team

1.  **Aussagekräftige Dateinamen:** Nutze statt `test.md` lieber `LF01_Hardware_Grundlagen.md`. Vermeide Leerzeichen in Dateinamen (nutze stattdessen Unterstriche `_`).
2.  **Vorschau nutzen:** Bevor du deine Änderungen speicherst ("committest"), schau dir immer die Vorschau an, um Formatierungsfehler zu vermeiden.
3.  **Bilder einbinden:** Wenn du Screenshots nutzt, erstelle am besten einen Ordner `assets/` oder `images/` im jeweiligen Verzeichnis, damit es ordentlich bleibt.
