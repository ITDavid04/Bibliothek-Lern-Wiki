5. Dein eduScrum-Lernkarten-Prompt einbauen (45 min)

Jetzt passt du die Ausgabe so an, dass nicht ein langer Bericht, sondern deine gewünschte Lernkarte entsteht. Dafür bearbeiten wir eine Vorlagen-Datei.
5.1 Öffne die Datei backend/report_type/custom_report.py

(Je nach Version kann die Struktur leicht abweichen – falls die Datei nicht existiert, schaue in backend/report_type/__init__.py oder die Doku. Zur Not erstelle eine neue Datei.)
5.2 Ersetze den Inhalt (bzw. suche die String-Variable für das Report-Template) durch folgenden Code:
python

from .research_report import generate_report  # Originalfunktion

CUSTOM_REPORT_TEMPLATE = """
Du bist ein erfahrener IT-Ausbilder, der komplexe Themen für Umschüler 
in agile eduScrum-Lernkarten gießt.

Erstelle zu folgendem Rechercheergebnis eine **Lernkarte** im einheitlichen Format:

**Titel des Themas**
**Kurze Essenz** (max. 3 verständliche Sätze)
**Lernziele** (2-3 Stück, als Aufzählung)
**Akzeptanzkriterien** (Woran merkt ein Lernender, dass er es verstanden hat?)
**Kernaussagen / Merksätze** (3-5 prägnante Punkte)
**Veranschaulichung** (eine Analogie oder ein Mini-Beispiel aus dem Alltag)
**Quellen** (max. 3 wichtigste URLs)

Halte die Sprache einfach, vermeide Füllwörter, orientiere dich an 
der Zielgruppe IT-Umschüler. Der Output soll direkt auf eine 
eduScrum-Karteikarte passen.

Rechercheergebnis:
{research_summary}
"""

def generate_custom_report(research_summary: str) -> str:
    # Hier wird der Prompt mit der Zusammenfassung gefüllt
    return CUSTOM_REPORT_TEMPLATE.format(research_summary=research_summary)

Wichtig: Die exakte Einbindung hängt von der GPT-Researcher-Version ab. Schaue in der Datei backend/report_type/__init__.py, wie andere Report-Typen registriert sind. Füge dort eine Zeile hinzu:
python

# In __init__.py, etwa Zeile 30 ergänzen:
elif report_type == "custom":
    from .custom_report import generate_custom_report
    report = generate_custom_report(research_summary)

(Falls das zu viel Programmierarbeit ist, können wir den Prompt auch direkt in die vorhandene research_report.py einbauen, aber dann verlierst du den Originalbericht. Ich empfehle den sauberen Weg.)

Alternative, falls dir das zu tief geht: Ändere einfach in config.py den REPORT_FORMAT = "research_report" und modifiziere die Datei backend/report_type/research_report.py. Suche nach der Variablen RESEARCH_REPORT_TEMPLATE und ersetze den langen Prompt durch unseren eduScrum-Prompt oben. Dann wird jeder generierte Bericht automatisch zur Lernkarte.

6. Für den täglichen Einsatz vorbereiten (30 min)

Damit du den Agenten bequem für beliebige Themen starten kannst, erstellen wir ein kleines Startskript.

Erstelle eine Datei recherche.py im Hauptverzeichnis von gpt-researcher:
python

# recherche.py – eduScrum-Recherche-Agent
import asyncio
from gpt_researcher import GPTResearcher

async def main():
    thema = input("🔍 Für welches Thema soll eine Lernkarte erstellt werden? ")
    print(f"\nRecherchiere '{thema}'... Bitte warten.\n")

    researcher = GPTResearcher(
        query=thema,
        report_type="custom",
        config_path="backend/config/config.py",
        verbose=False
    )
    report = await researcher.conduct_research()
    print("\n--- eduScrum-Lernkarte ---\n")
    print(report)

    with open("lernkarte.md", "w", encoding="utf-8") as f:
        f.write(report)
    print("\n✅ Lernkarte wurde gespeichert in 'lernkarte.md'.")

if __name__ == "__main__":
    asyncio.run(main())

Jetzt kannst du einfach im Terminal (virtuelle Umgebung aktiviert) ausführen:
bash

python recherche.py

und dein Thema eingeben. Die Ausgabe erscheint im Terminal und wird als Markdown-Datei gespeichert – perfekt zum Teilen mit deinen Kollegen im eduScrum-Board.
