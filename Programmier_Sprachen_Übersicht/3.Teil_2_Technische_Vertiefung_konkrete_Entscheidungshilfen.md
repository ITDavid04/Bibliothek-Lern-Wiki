2. Die technischen Steckbriefe – was unter der Haube passiert
🐍 Python – der gemütliche KI‑Gigant mit Turbo‑Update

    Stärken: Wahnsinns‑Bibliotheken, lesbar wie Englisch.

    Haken: Langsam wie eine Schnecke im direkten Vergleich. Deshalb ruft ernsthafter Python‑Code meist in C++ oder Rust geschriebene Module auf.

    Der Gamechanger 2026: Der Global Interpreter Lock (GIL) wird optional abschaltbar. Damit kann Python endlich mehrere CPU‑Kerne parallel nutzen. Aber Vorsicht: Alte C‑Erweiterungen können heimlich den GIL wieder einschalten und die Performance killen, ohne dass man es sofort merkt.

☕ Java – der Senior unter den Sprachen, der plötzlich Marathon läuft

    Das alte Problem: Jeder Web‑Request brauchte früher einen fetten Betriebssystem‑Thread (ca. 1 MB RAM). Bei 10.000 gleichzeitigen Nutzern war der Server am Ende.

    Die Lösung (Project Loom): Virtual Threads sind federleicht (wenige Kilobyte) und werden von Java selbst verwaltet. Plötzlich verträgt ein kleiner Server Millionen paralleler Verbindungen. Kombiniert mit GraalVM (vorab kompilierte native Binaries) starten Java‑Apps in Millisekunden – perfekt für Cloud‑Container.

    Aber Achtung: Ein synchronized‑Block kann einen Virtual Thread ausbremsen („Thread Pinning“). Das muss man wissen.

🐹 Go – der schlanke Cloud‑Handwerker

    Stärken: Kompiliert in Sekunden zu einer einzelnen Binary, die man einfach auf den Server kopiert. Goroutines machen Nebenläufigkeit so einfach wie nie.

    Haken: Kein „schöner“ Code. Fehlerbehandlung ist geschwätzig, Generics nur in Light‑Version. Dafür versteht jeder den Code auf Anhieb – das ist in großen Teams Gold wert.

🦀 Rust – der strenge, aber absolut vertrauenswürdige Bodyguard

    Stärken: Der Compiler ist wie ein unbestechlicher Prüfer: Er erwischt jeden Speicherfehler (Use‑after‑free, Buffer‑Overflow), bevor das Programm überhaupt startet. Und das ohne einen langsamen Garbage Collector.

    Haken: Die Lernkurve ist brutal. Man verbringt anfangs viel Zeit mit Kämpfen gegen den Compiler. Aber was einmal kompiliert, läuft stabil.

    Der Beweis aus der Praxis: Bei Google sank der Anteil speicherbedingter Android‑Sicherheitslücken durch Rust von 76 % (2019) auf unter 20 % (2025). In über 5 Millionen Zeilen Rust‑Code wurde vor Release nur eine einzige potentielle Speicher‑Schwachstelle gefunden.

3. Secure by Design – wenn der Staat zur Kasse bittet
Ausgangslage

C und C++ sind die Ursache der meisten kritischen Lücken. Das ist seit 30 Jahren bekannt.
Das Problem

Die US‑Cybersicherheitsbehörde CISA hat den 1. Januar 2026 als Deadline gesetzt. Firmen, die Software für kritische Infrastrukturen liefern, müssen öffentlich erklären, wie sie Speicherfehler‑Sprachen rauswerfen. Kein Plan = grobe Fahrlässigkeit im Schadensfall. Das kann existenzbedrohend werden.
Die Praxis

Empfohlen werden Rust, Go, Java, C#, Python, JavaScript. Besonders Rust wird massiv gepusht, weil es ohne Leistungsverlust C++ ersetzt. Selbst die DARPA (US‑Militärforschung) arbeitet an einer KI (TRACTOR), die automatisch Milliarden Zeilen C‑Code in sicheres, idiomatisches Rust übersetzen soll. Wer Rust kann, wird in Sicherheits‑ und Infrastrukturprojekten mit Gold aufgewogen.

6. KI im Entwickleralltag – der große Realitätscheck
Die Märchenstunde des Managements

84 % der Entwickler*innen nutzen KI‑Tools. Klingt super, oder?
Die Wahrheit aus den Umfragen (Stack Overflow 2025)

    46 % misstrauen dem KI‑Code, nur 33 % vertrauen ihm.

    Nur 14 % nutzen täglich KI‑Agenten – der Rest lehnt sie ab oder nutzt nur einfache Autovervollständigung.

    35 % aller Stack‑Overflow‑Besuche passieren, weil Leute von KI‑Assistenten verursachte Bugs reparieren müssen.

    „Vibe Coding“ (per Prompt Code generieren ohne Ahnung) führt zu instabilen Systemen und explodierenden technischen Schulden.

Was das für euch bedeutet

Der reine Code‑Schreiber wird ersetzbar. Gefragt ist der System‑Orchestrator, der KI‑Output prüft, Architekturfehler erkennt und für Sicherheit & Regelkonformität bürgt. Das könnt ihr lernen.
7. Legacy‑Modernisierung – wo die Musik spielt (30 Milliarden Dollar schwer)
Ausgangslage

70 % der Fortune‑500‑Unternehmen betreiben Systeme, die älter sind als die meisten Umschüler*innen.
Die versteckten Kosten

    10–15 veraltete Unternehmensanwendungen fressen pro Jahr 1,5–3,5 Mio. € (inkl. Sicherheitslücken, Spezialisten‑Zuschlägen, Workarounds).

    Windows‑10‑Extended‑Support kostet für 2.000 Geräte über 850.000 $ – nur für Sicherheitspatches.

    COBOL‑Spezialisten sind 40 % teurer und extrem schwer zu finden, dabei laufen 95 % aller Geldautomaten‑Transaktionen weltweit über COBOL‑Code (über 220 Milliarden Zeilen).

Die Rettung: API‑Led Modernization

Statt eines riskanten Totalabrisses (Big Bang) werden alte Systeme mit modernen Schnittstellen (z. B. TypeScript/Go) ummantelt. Neue Funktionen entstehen in modernen Sprachen, der alte Kern wird scheibchenweise ersetzt. Genau für diese Migrations‑Projekte werden Entwickler*innen mit Kenntnissen in modernen Sprachen händeringend gesucht.
Zusammenfassung für die IHK‑Prüfung

Wenn die Frage kommt: „Warum lohnt es sich, moderne Sprachen wie TypeScript, Go oder Rust zu lernen?“

    TypeScript macht große Webprojekte beherrschbar und hat JavaScript als aktive Entwicklungssprache abgelöst.

    Java bleibt dank Virtual Threads und GraalVM das unverwüstliche Enterprise‑Rückgrat.

    Go ist die Sprache der Cloud‑Infrastruktur und extrem einsteigerfreundlich.

    Rust wird regulatorisch erzwungen und eliminiert eine ganze Fehlerklasse – das schlägt sich direkt in Spitzengehältern nieder.

    Python ist die KI‑Schaltzentrale, aber zu langsam für Dauerlast; deshalb kombinieren Firmen es immer öfter mit Go oder Rust.

    Green IT und Secure‑by‑Design‑Vorgaben machen effiziente, sichere Sprachen zum betriebswirtschaftlichen Muss.

*Merksatz für die Prüfung: „Modern Coden heißt: TypeScript fürs Frontend, Python für KI und Go/Rust für alles, was schnell, sicher und energiesparend laufen muss – die goldenen Drei für das Jahr 2026.“*
