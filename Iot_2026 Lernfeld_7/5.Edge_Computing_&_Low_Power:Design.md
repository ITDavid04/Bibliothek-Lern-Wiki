## **1️⃣ 🌍 Grundlagen: Warum Edge Computing?**

*Die IHK will wissen: Wo liegt der Unterschied zur Cloud?*

### **📚 Kernkonzepte**

| **Konzept** | **Definition** | **IHK-relevante Frage** |
|-------------|---------------|-------------------------|
| **Edge Computing** | Datenverarbeitung **nahe am Entstehungsort** (z. B. Sensor, Gateway) statt in der Cloud. | *"Nennen Sie zwei Vorteile von Edge Computing gegenüber Cloud-Computing!"* |
| **Cloud Computing** | Zentrale Datenverarbeitung in Rechenzentren (z. B. AWS, Azure). | *"Wann ist Cloud-Computing trotzdem besser?"* |
| **Fog Computing** | Erweiterung von Edge Computing: **mehrere Edge-Geräte** bilden ein Netzwerk (z. B. Smart City). | *"Was ist der Unterschied zwischen Edge und Fog Computing?"* |

### **🔍 IHK-Prüfungsfragen (mit Musterlösungen)**

✅ **Frage:** *"Nennen Sie drei Vorteile von Edge Computing!"*
✔ **Antwort:**

1. **Geringere Latenz** (Echtzeitsteuerung, z. B. Industrie 4.0)
2. **Bandbreiteneinsparung** (Daten werden vorverarbeitet, nur relevante Infos werden gesendet)
3. **Ausfallsicherheit** (Funktioniert auch bei Netzwerkausfällen)
4. **Datenschutz** (Sensible Daten bleiben lokal)

✅ **Frage:** *"Wann ist Cloud-Computing trotzdem besser?"*
✔ **Antwort:**

- Bei **großen Datenmengen** (z. B. Videoüberwachung)
- Bei **komplexen Analysen** (z. B. KI-Modelle wie ChatGPT)
- Wenn **keine lokale Rechenleistung** verfügbar ist

## **2️⃣ ⚡ Low-Power Design: Energieeffizienz als Schlüssel**

*Für FISI: Hardware-Auswahl & Konfiguration | Für FIAE: Code-Optimierung*

### **📚 Kernkonzepte**

| **Maßnahme** | **Beschreibung** | **IHK-relevante Frage** |
|--------------|------------------|-------------------------|
| **Sleep-Modi** | Mikrocontroller in **Deep Sleep** (µA-Bereich) oder **Light Sleep** (ms-Aufwachzeit). | *"Wie reduziert man den Stromverbrauch eines ESP32?"* |
| **Taktfrequenz** | Reduzierte CPU-Taktung (z. B. 8 MHz statt 16 MHz). | *"Warum ist eine niedrigere Taktfrequenz energieeffizienter?"* |
| **Hardware-Auswahl** | Low-Power-Chips wie **ESP32-C3** oder **STM32L4** statt ESP8266. | *"Welcher Mikrocontroller eignet sich für batteriebetriebene IoT-Geräte?"* |
| **Datenübertragung** | QoS 0 statt QoS 2 (keine Bestätigungen = weniger Overhead). **Aber:** Nur für unkritische Sensordaten geeignet! | *"Welche QoS-Stufe spart am meisten Energie?"* |

### **🔍 IHK-Prüfungsfragen (mit Musterlösungen)**

✅ **Frage:** *"Nennen Sie drei Maßnahmen zur Reduzierung des Stromverbrauchs eines IoT-Geräts!"*
✔ **Antwort:**

1. **Deep Sleep nutzen** (z. B. `machine.deepsleep(5*60*1000000)` in MicroPython)
2. **Taktfrequenz reduzieren** (z. B. `setCpuFrequencyMhz(80)` für ESP32)
3. **Energiesparende Sensoren wählen** (z. B. BME280 statt DS18B20)
4. **QoS 0 für MQTT verwenden** (keine Bestätigungen = weniger Netzlast)

✅ **Frage:** *"Wie misst man den Stromverbrauch eines Mikrocontrollers?"*
✔ **Antwort:**

- **Multimeter** (für grobe Messungen)
- **Oszilloskop** (für präzise Stromverbrauchskurven)
- **Energy Monitoring Tools** (z. B. **J-Link Power Profiler** für ARM-Mikrocontroller)

### **🛠️ Code-Beispiel (FIAE: ESP32 mit Deep Sleep & MQTT)**

```python
# ESP32: Deep Sleep + MQTT-Publish (FIAE-relevant)
import machine
import network
import time
from umqtt.simple import MQTTClient

# WiFi-Konfiguration
WIFI_SSID = "dein_SSID"
WIFI_PASSWORD = "dein_Passwort"

# MQTT-Konfiguration
MQTT_BROKER = "test.mosquitto.org"
MQTT_TOPIC = "esp32/sensor/temperature"
MQTT_CLIENT_ID = "esp32_deep_sleep"

# Sensor (BME280)
from machine import I2C, Pin
import bme280

i2c = I2C(0, scl=Pin(22), sda=Pin(21), freq=400000)
bme = bme280.BME280(i2c=i2c)

# WiFi verbinden
def connect_wifi():
    sta_if = network.WLAN(network.STA_IF)
    if not sta_if.isconnected():
        sta_if.active(True)
        sta_if.connect(WIFI_SSID, WIFI_PASSWORD)
        while not sta_if.isconnected():
            time.sleep(0.5)
    print("WiFi verbunden:", sta_if.ifconfig())

# MQTT-Nachricht senden
def send_mqtt_message(temperature):
    client = MQTTClient(MQTT_CLIENT_ID, MQTT_BROKER)
    client.connect()
    client.publish(MQTT_TOPIC, str(temperature))
    client.disconnect()

# Hauptprogramm
def main():
    connect_wifi()
    bme.read_compensated_data()  # WICHTIG: Sensor auslesen!
    temperature = bme.temperature
    send_mqtt_message(temperature)
    print("Daten gesendet. Gehe in Deep Sleep...")
    machine.deepsleep(5 * 60 * 1000000)  # 5 Minuten

if __name__ == "__main__":
    main()
```

**🔹 Hinweis für FISI:**

- **Hardware:** ESP32-C3 (Low-Power) + BME280-Sensor
- **Konfiguration:** WiFi-Passwort und MQTT-Broker anpassen
- **Stromverbrauch:** Im Deep Sleep nur **~5 µA**!

## **3️⃣ 🔌 Edge-Architekturen: Protokolle & Hardware im Vergleich**

*Für FISI: Broker-Konfiguration | Für FIAE: Skriptentwicklung*

### **📚 Kernkonzepte**

| **Protokoll** | **Einsatzgebiet** | **Vorteile** | **Nachteile** | **IHK-relevante Frage** |
|---------------|-------------------|--------------|---------------|-------------------------|
| **MQTT** | Echtzeit-Datenübertragung (IoT) | Leichtgewichtig, zuverlässig (QoS), Pub/Sub | Keine großen Payloads | *"Warum ist MQTT ideal für IoT?"* |
| **CoAP** | Ressourcenbeschränkte Geräte (Low-Power) | UDP-basiert, ideal für Low-Power | **Overhead durch Bestätigungen bei CON, keine Zustellgarantie bei NON** | *"Wann wird CoAP statt MQTT verwendet?"* |
| **OPC UA** | Industrieautomation | Standardisiert, typsicher, Security-Profile | **Hoher Overhead** (by design für Sicherheit) | *"Warum hat OPC UA mehr Overhead als MQTT?"* |
| **LoRaWAN** | Weitverkehrsnetzwerke | Geringer Stromverbrauch, große Reichweite | Geringe Datenrate | *"Für welche Anwendungen eignet sich LoRaWAN?"* |

### **🔍 IHK-Prüfungsfragen (mit Musterlösungen)**

✅ **Frage:** *"Welches Protokoll eignet sich für ein batteriebetriebenes IoT-Gerät mit geringer Datenrate?"*
✔ **Antwort:**

- **LoRaWAN** (geringer Stromverbrauch, große Reichweite)
- **CoAP** (UDP-basiert, ideal für Low-Power)

✅ **Frage:** *"Warum hat OPC UA mehr Overhead als MQTT?"*
✔ **Antwort:**

- OPC UA ist **typsicher** (binäre Kodierung) und unterstützt **Security-Profile** (z. B. Verschlüsselung).
- MQTT ist **einfach und leichtgewichtig** (nur Pub/Sub, keine Typsicherheit).

### **🛠️ Code-Beispiel (FIAE: CoAP-Client mit Python & aiocoap)**

```python
# CoAP-Client für Low-Power-Anwendungen (FIAE-relevant)
# Installiere zuerst: pip install aiocoap
import asyncio
from aiocoap import Context, Message, GET

async def coap_request():
    context = await Context.create_client_context()

    # CoAP-Server (z. B. lokal oder public: coap://coap.me)
    request = Message(code=GET, uri="coap://coap.me/test")

    response = await context.request(request).response
    print("CoAP Response:", response.payload.decode())

# Hauptprogramm
asyncio.run(coap_request())
```

**🔹 Hinweis für FISI:**

- **CoAP-Server einrichten:** z. B. mit **Californium** (Java) oder **CoAPthon** (Python).
- **Security:** CoAP unterstützt **DTLS** (ähnlich wie TLS für UDP).

## **4️⃣ 🔐 Sicherheit in Edge-Netzwerken**

*Für FISI: Broker-Konfiguration & Zertifikate | Für FIAE: Code mit TLS*

### **📚 Kernkonzepte**

| **Bedrohung** | **Gegenmaßnahme** | **IHK-relevante Frage** |
|---------------|-------------------|-------------------------|
| **Unverschlüsselte Kommunikation** | **TLS/SSL** (MQTT über Port 8883) | *"Wie sichert man MQTT vor MITM-Angriffen?"* |
| **Schwache Authentifizierung** | **Zertifikatsbasierte Authentifizierung** | *"Warum sind Standard-Passwörter gefährlich?"* |
| **Veraltete Firmware** | **OTA-Updates** (Over-the-Air) | *"Wie aktualisiert man die Firmware eines Edge-Geräts?"* |
| **Physikalischer Zugriff** | **Secure Boot** (Hardware-Security) | *"Wie schützt man ein Edge-Gerät vor Manipulation?"* |

### **🔍 IHK-Prüfungsfragen (mit Musterlösungen)**

✅ **Frage:** *"Wie sichert man MQTT vor MITM-Angriffen?"*

✔ **Antwort:**

1. **TLS/SSL aktivieren** (MQTT über Port 8883)
2. **Zertifikatsbasierte Authentifizierung** (MQTT-Client mit eigenem Zertifikat)
3. **MQTT-Broker (z. B. Mosquitto) mit TLS konfigurieren**

✅ **Frage:** *"Warum sind Standard-Passwörter gefährlich?"*

✔ **Antwort:**

- **Einfache Angriffe** (z. B. Brute-Force)
- **Keine Zugriffskontrolle** (jeder mit "admin:admin" kann das Gerät steuern)
- **Compliance-Probleme** (DSGVO, ISO 27001)

### **🛠️ Code-Beispiel (FIAE: MQTT mit TLS auf ESP32)**

```python
# ESP32: MQTT mit TLS (FIAE-relevant)
import network
import time
from umqtt.simple import MQTTClient
import machine
import ssl

# WiFi-Konfiguration
WIFI_SSID = "dein_SSID"
WIFI_PASSWORD = "dein_Passwort"

# MQTT-Konfiguration (TLS!)
MQTT_BROKER = "broker.hivemq.com"  # Public Broker mit TLS
MQTT_PORT = 8883
MQTT_TOPIC = "esp32/sensor/temperature"
MQTT_CLIENT_ID = "esp32_tls"

# Root-CA-Zertifikat (korrekt extrahiert!)
ROOT_CA = """-----BEGIN CERTIFICATE-----
MIIFazCCBFOgAwIBAgISA... (Hier Root-CA einfügen)
-----END CERTIFICATE-----"""

# WiFi verbinden
def connect_wifi():
    sta_if = network.WLAN(network.STA_IF)
    if not sta_if.isconnected():
        sta_if.active(True)
        sta_if.connect(WIFI_SSID, WIFI_PASSWORD)
        while not sta_if.isconnected():
            time.sleep(0.5)
    print("WiFi verbunden:", sta_if.ifconfig())

# MQTT-Nachricht senden (mit TLS)
def send_mqtt_message_tls(temperature):
    client = MQTTClient(
        client_id=MQTT_CLIENT_ID,
        server=MQTT_BROKER,
        port=MQTT_PORT,
        ssl=True,
        ssl_params={
            "cert_reqs": ssl.CERT_REQUIRED,  # Robuster als numerischer Wert
            "ca_certs": ROOT_CA
        }
    )
    client.connect()
    client.publish(MQTT_TOPIC, str(temperature))
    client.disconnect()

# Hauptprogramm
def main():
    connect_wifi()
    temperature = 23.5  # Beispielwert
    send_mqtt_message_tls(temperature)
    print("Daten gesendet (TLS). Gehe in Deep Sleep...")
    machine.deepsleep(5 * 60 * 1000000)  # 5 Minuten

if __name__ == "__main__":
    main()
```

**🔹 Hinweis für FISI:**

- **Mosquitto-Broker mit TLS einrichten:**

  ```bash
  sudo apt install mosquitto mosquitto-clients
  sudo nano /etc/mosquitto/mosquitto.conf
  ```

  ```ini
  listener 8883
  protocol mqtt
  cafile /etc/mosquitto/certs/ca.crt
  certfile /etc/mosquitto/certs/server.crt
  keyfile /etc/mosquitto/certs/server.key
  ```

- **Zertifikate korrekt generieren:**

  ```bash
  # Root-CA erstellen
  openssl req -new -x509 -days 365 -nodes -out ca.crt -keyout ca.key

  # Server-Zertifikat erstellen
  openssl genrsa -out server.key 2048
  openssl req -new -out server.csr -key server.key
  openssl x509 -req -in server.csr -CA ca.crt -CAkey ca.key -CAcreateserial -out server.crt -days 365

  # Root-CA extrahieren (nur das Root-Zertifikat!)
  openssl s_client -showcerts -connect broker.hivemq.com:8883 </dev/null 2>/dev/null \
  | awk '/BEGIN CERTIFICATE/,/END CERTIFICATE/{if(++m==2)print;if(/END CERTIFICATE/)m=0}' > root_ca.pem
  ```

### **🛠️ Code-Beispiel (FIAE: Echte Gesichtserkennung mit BlazeFace auf Raspberry Pi)**

```python
# Raspberry Pi: Echte Gesichtserkennung mit BlazeFace (FIAE-relevant)
import cv2
import numpy as np
import tflite_runtime.interpreter as tflite

# Modell laden (BlazeFace von Google)
interpreter = tflite.Interpreter(model_path="blazeface_float.tflite")
interpreter.allocate_tensors()

# Kamera initialisieren
cap = cv2.VideoCapture(0)

while True:
    ret, frame = cap.read()
    if not ret:
        break

    # Bild vorverarbeiten (256x256 für BlazeFace)
    input_data = cv2.resize(frame, (256, 256))
    input_data = np.expand_dims(input_data, axis=0).astype(np.float32)

    # Inferenz durchführen
    interpreter.set_tensor(interpreter.get_input_details()[0]['index'], input_data)
    interpreter.invoke()
    boxes = interpreter.get_tensor(interpreter.get_output_details()[0]['index'])[0]
    scores = interpreter.get_tensor(interpreter.get_output_details()[1]['index'])[0]

    # Gesichter zeichnen
    for box, score in zip(boxes, scores):
        if score > 0.5:  # Mindest-Confidence
            ymin, xmin, ymax, xmax = box
            xmin = int(xmin * frame.shape[1])
            xmax = int(xmax * frame.shape[1])
            ymin = int(ymin * frame.shape[0])
            ymax = int(ymax * frame.shape[0])
            cv2.rectangle(frame, (xmin, ymin), (xmax, ymax), (0, 255, 0), 2)

    cv2.imshow("Edge AI - Echte Gesichtserkennung", frame)

    if cv2.waitKey(1) & 0xFF == ord('q'):
        break

cap.release()
cv2.destroyAllWindows()
```

**🔹 Hinweis für FISI:**

- **Hardware:** Raspberry Pi 4 + Kamera-Modul
- **Modell:** BlazeFace (optimiert für Edge-Geräte)
- **Stromverbrauch:** ~2-3W (im Vergleich zu Cloud: ~100W für GPU)
