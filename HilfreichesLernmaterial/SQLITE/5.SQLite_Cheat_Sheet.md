### 🧠 5 Merksätze – worum es bei SQLite wirklich geht

1. **SQLite ist serverlos.** Keine Installation, keine Konfiguration, keine Administration. Die gesamte Datenbank ist eine einzige Datei.
2. **Nicht die Spalte bestimmt den Typ, sondern der Wert.** (Manifest Typing) – maximale Flexibilität, aber in der Prüfung aufpassen.
3. **SQLite ist eine der am härtesten getesteten Softwarekomponenten der Welt.** 92 Mio. Zeilen Testcode, 100 % Branch-Coverage, zertifiziert für die Luftfahrt.
4. **Ab 2024–2026 kann SQLite auch Cloud, Browser und Hochleistung.** JSONB, WASM + OPFS und schnelle SSDs haben die Regeln geändert.
5. **SQLite ist die am weitesten verbreitete Datenbank der Erde.** Über 1 Billion Instanzen – in deinem Handy, deinem Browser, in Flugzeugen und auf dem Mars.

### 🖥️ SQLite Shell: Die 10 wichtigsten dot-Befehle

| Befehl | Wirkung | Wann du ihn brauchst |
|--------|---------|----------------------|
| `.tables` | Alle Tabellen anzeigen | Erste Orientierung nach dem Öffnen |
| `.schema tabelle` | CREATE TABLE der Tabelle zeigen | Schema prüfen, bevor du änderst |
| `.mode csv` | Ausgabe auf CSV umstellen | Vor Export oder Import |
| `.separator ";"` | Trennzeichen ändern | Bei deutschen CSV-Dateien (Semikolon) |
| `.import datei.csv tabelle` | CSV-Datei in Tabelle einlesen | Datenmigration aus Excel |
| `.headers on` | Spaltennamen in der Ausgabe zeigen | Deutlich bessere Lesbarkeit |
| `.output datei.txt` | Ausgabe in Datei umleiten | Ergebnisse speichern |
| `.dump` | Gesamte DB als SQL-Skript exportieren | Backup oder Migration |
| `.read skript.sql` | SQL-Skript ausführen | Migrationen einspielen |
| `.quit` | SQLite beenden | Raus aus der Shell |

**Starten:** `sqlite3 meine_db.db`  
**Hilfe:** `.help`

### 📊 Datentypen & Typ-Affinität

SQLite kennt 5 **Typ-Affinitäten** (Empfehlungen für die Spalte – nicht harte Regeln):

| Affinität | Wird bevorzugt bei Deklaration mit | Beispiel | Gespeichert als |
|-----------|------------------------------------|----------|-----------------|
| **INTEGER** | `INT`, `INTEGER`, `TINYINT`, `BIGINT` | `ALTER TABLE t ADD COLUMN i INT` | Ganze Zahl |
| **TEXT** | `TEXT`, `CHAR`, `VARCHAR`, `CLOB` | `name TEXT` | Text (UTF-8) |
| **REAL** | `REAL`, `FLOAT`, `DOUBLE` | `preis REAL` | 8-Byte-Gleitkommazahl |
| **BLOB** | `BLOB` oder keine Affinität erkennbar | `bild BLOB` | Binärdaten (exakt so, wie eingefügt) |
| **NUMERIC** | `NUMERIC`, `DECIMAL`, `BOOLEAN`, `DATE` | `gehalt NUMERIC` | INTEGER oder REAL, je nach Wert |

> 🎯 **Prüfungstipp:** In der schriftlichen Prüfung immer **statische Typen** verwenden (`INT`, `VARCHAR(n)`, `DECIMAL(p,s)`, `DATE`) – so wie MySQL oder Oracle es erwarten.

### 🏗️ DDL Quick Reference – Tabellen anlegen mit allen Constraints

**Standardschablone (SQLite-Dialekt):**
```sql
CREATE TABLE tabellenname (
    spaltenname  TYP [CONSTRAINTS],
    ...
);
```

| Constraint | Syntax | Bedeutung |
|------------|--------|------------|
| **PRIMARY KEY** | `id INTEGER PRIMARY KEY AUTOINCREMENT` | Eindeutige ID, zählt automatisch |
| **FOREIGN KEY** | `spalte INT REFERENCES zieltabelle(zielspalte)` | Verknüpfung zu anderer Tabelle |
| **NOT NULL** | `name TEXT NOT NULL` | Darf niemals leer sein |
| **UNIQUE** | `email TEXT UNIQUE` | Jeder Wert nur einmal erlaubt |
| **CHECK** | `preis REAL CHECK (preis > 0)` | Eigene Bedingung |
| **DEFAULT** | `erstellt TEXT DEFAULT (datetime('now'))` | Automatischer Standardwert |

**Beispiel mit allen Constraints:**
```sql
CREATE TABLE mitarbeiter (
    ma_id   INTEGER PRIMARY KEY AUTOINCREMENT,
    name    TEXT NOT NULL,
    email   TEXT UNIQUE NOT NULL,
    gehalt  REAL CHECK (gehalt >= 0),
    abt_id  INTEGER REFERENCES abteilung(abt_id),
    angelegt TEXT DEFAULT (datetime('now'))
);
```

### 🔍 DML für die Prüfung – die häufigsten Abfragemuster

| Aufgabe | Grundgerüst |
|---------|-------------|
| **Alle Spalten, alle Zeilen** | `SELECT * FROM tabelle;` |
| **Bestimmte Spalten mit Bedingung** | `SELECT sp1, sp2 FROM t WHERE sp3 = 'Wert';` |
| **Sortiert** | `SELECT * FROM t ORDER BY sp ASC;` (ASC = aufsteigend, DESC = absteigend) |
| **Eindeutige Werte** | `SELECT DISTINCT sp FROM t;` |
| **Mit INNER JOIN (n:m) aufgelöst über Assoziationstabelle** | `SELECT a.name, m.titel FROM autor a JOIN autor_medium am ON a.id = am.autor_id JOIN medium m ON am.medium_id = m.id;` |
| **Mit LEFT JOIN (auch Treffer ohne Partner zeigen)** | `SELECT k.name, b.titel FROM kunde k LEFT JOIN bestellung b ON k.id = b.kunde_id;` |
| **Gruppiert mit Aggregatfunktion** | `SELECT abteilung, COUNT(*) AS anzahl FROM mitarbeiter GROUP BY abteilung;` |
| **Gruppiert mit Filter danach (HAVING)** | `SELECT abteilung, AVG(gehalt) FROM mitarbeiter GROUP BY abteilung HAVING AVG(gehalt) > 3000;` |
| **Unterabfrage mit IN** | `SELECT name FROM kunde WHERE id IN (SELECT kunde_id FROM bestellung WHERE betrag > 500);` |
| **Daten ändern** | `UPDATE t SET sp = wert WHERE bedingung;` |
| **Daten löschen** | `DELETE FROM t WHERE bedingung;` |

> 🎯 **Prüfungstipp:** 90 % der schriftlichen SQL-Aufgaben sind `SELECT`-Abfragen. Wenn du die oberen 7 Zeilen dieser Tabelle im Schlaf beherrschst, hast du die halbe Miete. Denk an die Reihenfolge: `SELECT` → `FROM` → `JOIN` → `WHERE` → `GROUP BY` → `HAVING` → `ORDER BY`.

### 📥 Import / Export in einer Zeile

| Aufgabe | Befehl |
|---------|--------|
| **CSV importieren (Komma)** | `sqlite3 db.db ".mode csv" ".import daten.csv tabelle"` |
| **CSV importieren (Semikolon)** | `sqlite3 db.db ".mode csv" ".separator ;" ".import daten.csv tabelle"` |
| **Ganze DB als SQL exportieren (Backup)** | `sqlite3 db.db ".dump" > backup.sql` |
| **SQL-Dump wieder einspielen** | `sqlite3 neue_db.db < backup.sql` |

### ⚖️ IHK-Spicker: Die 5 wichtigsten Dialekt-Unterschiede SQLite ↔ MySQL

| # | Konzept | SQLite | MySQL / Standard-Prüfung |
|---|---------|--------|--------------------------|
| 1 | **Auto-ID** | `INTEGER PRIMARY KEY AUTOINCREMENT` | `INT AUTO_INCREMENT PRIMARY KEY` |
| 2 | **Boolean** | `INTEGER` (0/1) | `BOOLEAN` oder `TINYINT(1)` |
| 3 | **Datum als Typ** | Gibt's nicht; nutze `TEXT` mit `date()` | `DATE`, `DATETIME` |
| 4 | **String-Verkettung** | `'A' \|\| 'B'` | `CONCAT('A', 'B')` (MySQL) |
| 5 | **FULL/RIGHT JOIN** | `FULL OUTER JOIN` erst ab 3.39, kein `RIGHT JOIN` | Beide Standard |

**Für die schriftliche Prüfung:** Immer MySQL-Syntax üben! Im Projekt darfst du SQLite-Dialekt nutzen, musst es aber sauber dokumentieren.
