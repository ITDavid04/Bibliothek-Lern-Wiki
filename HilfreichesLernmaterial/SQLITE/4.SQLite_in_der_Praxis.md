### 🧱 Der schnellste Einstieg: SQLite von null auf Tabelle in 5 Minuten

Weil SQLite ohne Installation auskommt, ist der Start so einfach wie bei kaum einer anderen Technologie. Alles, was du brauchst, ist die Kommandozeilen-Shell `sqlite3`.

```bash
# 1. Terminal öffnen und eine neue Datenbankdatei starten
sqlite3 meine_projektdatenbank.db

# 2. Erste Tabelle anlegen
CREATE TABLE kunden (
    kunden_id INTEGER PRIMARY KEY AUTOINCREMENT,
    name TEXT NOT NULL,
    email TEXT UNIQUE
);

# 3. Einen Datensatz einfügen
INSERT INTO kunden (name, email) VALUES ('Max Mustermann', 'max@example.com');

# 4. Abfrage testen
SELECT * FROM kunden;

# 5. SQLite beenden
.quit
```

Fertig. Kein Server-Start, keine Passworteingabe, keine Port-Konfiguration. Die Datei `meine_projektdatenbank.db` liegt in deinem aktuellen Verzeichnis und enthält alle Daten.

**Die wichtigsten dot-Befehle für die Shell:**

| Befehl | Wirkung | Nutzen |
|--------|---------|--------|
| `.tables` | Zeigt alle Tabellen der Datenbank | Erste Orientierung |
| `.schema tabellenname` | Zeigt das CREATE TABLE einer Tabelle | Überprüfen, ob die Struktur stimmt |
| `.mode csv` | Schaltet Ausgabe auf CSV | Ideal für Export |
| `.import datei.csv tabelle` | Importiert CSV direkt | Siehe nächstes Kapitel |
| `.headers on` | Zeigt Spaltennamen in Abfragen | Deutlich besser lesbar |
| `.output datei.txt` | Leitet Ausgabe in Datei | Für Berichte und Logs |
| `.dump` | Exportiert gesamte DB als SQL | Backup und Migration |
| `.read datei.sql` | Führt SQL-Skript aus | Für Migrationen und Tests |

### 📋 Der CSV-Import: Aus Excel in die relationale Welt

Ein Klassiker im Projektalltag: Der Kunde liefert Daten als Excel- oder CSV-Datei, und du sollst sie in deine Anwendung überführen. SQLite macht das zum Kinderspiel – komplett ohne externe Tools.

**Schritt für Schritt: `kunden.csv` importieren**

```bash
# 1. Datenbank öffnen
sqlite3 mein_projekt.db

# 2. Zieltabelle vorbereiten (Spalten müssen mit CSV übereinstimmen!)
CREATE TABLE kunden (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    name TEXT,
    email TEXT,
    ort TEXT
);

# 3. Import-Modus setzen und CSV einlesen
.mode csv
.import kunden.csv kunden

# 4. Prüfen, ob alle Daten da sind
SELECT COUNT(*) FROM kunden;
SELECT * FROM kunden LIMIT 5;
```

**Typische Fallstricke und Lösungen:**

| Problem | Lösung |
|---------|--------|
| Erste Zeile der CSV enthält Spaltenüberschriften | `.import` fügt die trotzdem als Daten ein. Abhilfe: Vor Import `--skip 1` oder mit einem Skript die erste Zeile abschneiden |
| Trennzeichen ist nicht Komma, sondern Semikolon (deutsches Excel) | `.mode csv` erwartet Komma. Abhilfe: `.separator ";"` vor `.import` setzen |
| Datumswerte werden als Text interpretiert | In der Zieltabelle Datumsspalte als `TEXT` definieren, SQLite kann mit `date()` darauf arbeiten |
| NULL-Werte in der CSV | Leere Felder werden als leere Strings importiert. Abhilfe: `UPDATE t SET sp = NULL WHERE sp = '';` |

> 💡 **Agile Side-Info:** Im Sprint hast du oft den Fall, dass der Product Owner Testdaten als Excel schickt. Statt stundenlang ein Import-Tool zu bauen, nutzt du die `.import`-Funktion für den Prototyp – in 2 Minuten sind die Daten drin, und du kannst sofort mit der Entwicklung der eigentlichen Features loslegen.

### 🖥️ Die besten GUI-Tools für SQLite 2026

Nicht jeder steht auf schwarze Bildschirme. Diese grafischen Werkzeuge sind für Einsteiger und Fortgeschrittene gleichermaßen empfehlenswert:

| Tool | Preis | Besonderheit | Ideal für |
|------|-------|--------------|-----------|
| **DB Browser for SQLite (DB4S)** | Kostenlos, Open Source | Tabellenansicht wie Excel, Assistenten für Tabellenerstellung, ab 04/2026 auch Windows on ARM | Einsteiger, die visuell arbeiten wollen |
| **DBeaver Community** | Kostenlos | Universell: Kann SQLite, MySQL, PostgreSQL und viele mehr | Fortgeschrittene, die mehrere DB-Typen nutzen |
| **SQLiteStudio** | Kostenlos | Umfangreiche Schema-Design-Werkzeuge, Plugin-System | Entwickler, die oft am Schema arbeiten |
| **SQLMate (macOS)** | Kostenpflichtig (ca. 20 €) | Modern, native SwiftUI, sehr schnell, schöne Visualisierungen | macOS-Nutzer mit Designanspruch |
| **Beekeeper Studio** | Community Edition kostenlos | Cleanes UI, Query-History, dunkles Design | Alle, die ein modernes Tool suchen |

**Empfehlung für den Einstieg:** Installiere dir DB Browser for SQLite und DBeaver parallel. DB4S ist perfekt, um schnell Tabellen zu sehen oder per Assistenten eine neue Spalte hinzuzufügen. DBeaver begleitet dich dann, wenn du später mit mehreren Datenbanktypen arbeitest und ein professionelles Tool brauchst.

### 💎 Nice to Know: Kuriositäten und Insiderwissen

Diese Fakten machen sich nicht nur gut im Smalltalk mit Kollegen und Prüfern, sondern zeigen, dass du dich wirklich mit der Technologie auseinandergesetzt hast:

**⚓ Geburt auf hoher See**  
D. Richard Hipp begann die Entwicklung von SQLite im Jahr 2000 auf einem US-Zerstörer (USS Oscar Austin). Das Ziel: eine Datenbank, die ohne Server-Administration funktioniert – weil man mitten auf dem Ozean keinen DBA anrufen kann.

**📜 Der Segen im Quellcode**  
Jede SQLite-Quelldatei beginnt nicht mit einer Lizenz, sondern mit einem Segen: *„May you do good and not evil. May you find forgiveness for yourself and forgive others. May you share freely, never taking more than you give.“*

**🧘 Code of Ethics statt Code of Conduct**  
Statt eines typischen Verhaltenskodex verwendet das SQLite-Projekt einen „Code of Ethics“, der auf der Regel des Heiligen Benedikt basiert. Die Kernwerte: Redlichkeit, Bescheidenheit und Respekt.

**⚡ Schneller als das Dateisystem**  
In bestimmten Szenarien ist SQLite bis zu 35 % schneller als direkte `fopen()`-Aufrufe, weil es die Anzahl der Systemaufrufe minimiert. Die Datenbank schlägt das Betriebssystem auf seinem eigenen Terrain.

**🦖 Abwärtskompatibilität über Jahrzehnte**  
SQLite garantiert, dass Datenbankdateien von heute auch in 30 Jahren noch lesbar sind. Bestimmte Bugs werden **bewusst nicht gefixt**, um die Kompatibilität mit Milliarden existierender Dateien nicht zu gefährden.

### 🚀 Im Stil des agilen Arbeitens

- **DB-Dateien in die Versionsverwaltung?** Für die Entwicklung: Ja! Eine kleine Testdatenbank mit Beispieldaten kann ins Git-Repo. So können neue Teammitglieder sofort loslegen, ohne erst ein Import-Skript auszuführen.  
- **Automatisierte CSV-Import-Tests:** Schreibe ein kleines Shell-Skript, das regelmäßig (z. B. im CI/CD-Pipeline) die gleichen CSV-Daten importiert und mit `SELECT COUNT(*)` prüft, ob alle Zeilen angekommen sind. So sicherst du die Importqualität ab.  
- **Tool-Wechsel zulassen:** Im Team darf jeder das GUI-Tool nutzen, das ihm liegt – DB Browser, DBeaver oder die Shell. Einzige Regel: Alle Änderungen müssen als SQL-Skript gespeichert und nachvollziehbar sein.

🎯 **Side-Info:** Profis nutzen die Shell für alles, was in Skripten wiederholbar sein muss, und das GUI-Tool für Exploration und Ad-hoc-Analyse. Beide Welten zu beherrschen, macht dich schnell und unabhängig.
